# Enterprise Banking Platform

This repository contains a production-grade Spring Boot-based digital banking platform.

## Prerequisites

- Java 21
- Maven 3.8+
- Docker & Docker Compose
- (Optional) Redis, PostgreSQL, Kafka if not running via Docker Compose

## Local development (Docker Compose)

1. Build the application (skip tests to speed up):

```bash
mvn -DskipTests package
```

2. Start services with Docker Compose:

```bash
docker-compose up --build
```

3. Access the API:

- Swagger UI: http://localhost:8080/docs/swagger
- API root: http://localhost:8080/api/v1

## Run tests

Run unit and integration tests (requires Docker for Testcontainers):

```bash
mvn -B clean verify
```

## Database migrations

Migrations are managed by Flyway under `src/main/resources/db/migration`.

To run migrations manually against a DB:

```bash
mvn flyway:migrate -Dflyway.url=jdbc:postgresql://localhost:5432/banking -Dflyway.user=bank_user -Dflyway.password=bank_pass
```

## Production checklist

- Rotate and store secrets in a secrets manager (do **not** use `application.yml` for production secrets).
- Configure TLS termination (load balancer or ingress) and enforce HTTPS.
- Use Redis for token blacklist and caching (already supported).
- Configure logging (centralized ELK/CloudWatch) and metrics (Prometheus/Grafana).
- Configure backups and disaster recovery for PostgreSQL.
- Run security scans and dependency vulnerability checks.
- Add health checks and readiness probes for orchestration.
- Run load tests to tune connection pools and database indices.

## CI/CD

The repository contains a GitHub Actions workflow to build, test, and build Docker images; ensure runners have Docker available for integration tests.

## Notes

- The app uses JWT access and refresh tokens, Redis for revocation, Flyway migrations, and Testcontainers for integration tests.
- For scaling, migrate token revocation and rate limiting to a distributed cache and consider outbox pattern for reliable event publishing.
